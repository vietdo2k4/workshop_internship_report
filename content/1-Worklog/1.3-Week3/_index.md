### Week 3 Objectives:

* Understand the role of architecture diagrams when presenting a system on AWS.
* Learn how to read and sketch AWS diagrams using basic layers.
* Use AWS Architecture Icons when drawing architecture diagrams.
* Understand the main ideas of the AWS Well-Architected Framework.
* Practice a short serverless lab to understand how API Gateway connects to Lambda.

### Week 3 Results:

* Learned how to read AWS diagrams through key layers: client, frontend, network, backend, storage/database, and monitoring.

* Created a simple web architecture diagram using AWS Architecture Icons.

* Understood the 6 pillars of the AWS Well-Architected Framework and why architecture should not only “work”.

* Completed a basic serverless lab with API Gateway and Lambda, then checked the response and logs.

* Distinguished the basic roles of RDS and DynamoDB when placing a database layer in a diagram.

* Built the foundation for Week 4, which focuses on workshop writing, report structure, and clearer architecture documentation.
